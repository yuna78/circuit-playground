# 电路乐园 MVP 技术设计

## Context

全新空仓库，从零搭建。产品形态：纯前端中文网页游戏，教初中（+高中入门）电学。核心闭环：拖元件 → 连线 → 实时解算 → 可视化电流/电压 → 改动参数看变化。用户为家长与孩子，运行环境为桌面/平板现代浏览器，部署为 Cloudflare Pages 静态站点。

参考项目：PhET circuit-construction-kit-dc（交互设计对照）、Falstad circuitjs1（解算器对照）。两者均可作为"物理正确性"的对拍标准。

## Goals / Non-Goals

**Goals:**

- 任意接法都能算、不崩溃、有物理正确的反馈（短路发烫、保险丝烧断、断路无流）
- 电流/电压"看得见"：小点流动 + 电势颜色梯度 + 元件状态（亮度/发热/烧断）
- 沙盒 + 8 章闯关双模式，共用同一画布与解算器
- 解算器为独立纯函数模块，可单测、可与 Falstad/手算对拍
- 全中文，术语对齐人教版初中物理
- 零后端、零运行成本，`npm run build` 产物直接上 Cloudflare Pages

**Non-Goals:**

- 交流电、电容/电感暂态、二极管等非线性元件（MVP 只做直流阻性电路）
- 手机小屏适配（平板/桌面优先）
- 账号、云存档、多人协作、排行榜
- 关卡编辑器（关卡由 JSON 手写；编辑器留待后续 change）
- 音效与背景音乐（后续 change 可加）

## Decisions

### D1. 技术栈：Vite + React 19 + TypeScript + Zustand（脚手架实际产出 React 19 + oxlint，向下兼容，照用）

- React 生态最熟、组件化 UI（抽屉/面板/对话框）效率最高；画布内部不依赖 React 重渲染管线（见 D3）。
- Zustand 做全局状态（电路文档、解算结果、关卡进度）：比 Redux 轻，selector 订阅粒度细，画布高频更新不会拖垮 UI。
- 备选 Svelte：包更小，但生态与后续维护性（Claude/社区代码参考量）不如 React。
- 无 UI 组件库依赖：界面元素少且全定制卡通风，手写 + CSS 即可，避免风格打架。

### D2. 解算器：自研 MNA（改进节点分析），数值库仅需高斯消元

- 电路规模 ≤ 数十节点，手写高斯消元（部分主元）足够，无需引入 math.js 等重依赖。
- MNA 标准形：电导矩阵 G 填电阻类元件，电压源（电池含内阻 r，MVP 初中关卡 r=0，高中章节 r>0）用扩展电流变量行。
- **理想导线/闭合开关** 建模为"节点合并"（union-find 把导线两端并成同一电学节点），而不是 1e-9Ω 小电阻——避免病态矩阵，短路判定也更干净。**理想电流表改为 0V 电压源支路**（MNA 电压源的电流变量即读数，精确且免后处理）。
- **导线/开关上的电流**（供小点动画用）：节点合并会丢失单根导线电流，解算后做一次"簇内流网络"后处理——同一电学节点内，端子注入电流已知（来自各元件支路电流），对导线图取生成树解流量守恒，环路导线电流置 0（任意分配均满足 KCL，取 0 最稳定）。
- **断路/断开开关/烧断的保险丝**：从图中移除该支路。孤立子图各自独立解（或电势置为参考不定值，UI 显示"未接通"）。
- **短路检测**：电压源两端被合并为同一节点 → 直接判定"电池短路"，不解方程，触发教学化反馈（电池发烫警告动画 + 提示文案）。这是初中重点错误场景，值得特判。
- **电压表**：建模为大内阻（10MΩ）支路，读数 = 两端电势差——保持"电压表并联、内阻很大"的课本模型，高中章节还能教"电表不理想"。
- 保险丝：解算后检查 I > 额定值 → 标记熔断 → 状态写回电路文档 → **重新解算一次**（熔断改变拓扑）。灯泡烧毁同理（超额定功率）。
- 解算触发：电路拓扑/参数变化时同步重解（毫秒级）；滑动变阻器拖动时逐帧重解，天然实现"动态分析"演示。
- 备选：移植 Falstad 解算器——GPL 传染 + Java 转译代码难维护，不如自研 300 行干净模块。**开发时用 Falstad/手算做对拍单测**。

### D3. 画布：SVG 渲染 + 网格吸附，React 只管元件树、动画走 rAF 直改 DOM

- SVG 而非 Canvas：元件是矢量卡通素材，SVG 天然支持缩放不糊、CSS 滤镜（发光/发热）、事件命中（点选元件）。元件数量少（<50），无性能压力。
- 电流小点动画：预计算每条导线路径（SVG path），小点用 `requestAnimationFrame` 按电流大小推进 offset，直接改 DOM attribute，不经 React——避免 60fps 的 re-render。
- 网格吸附连线模型：**元件占格 + 端点在格点上，导线 = 正交折线段**（类似 PhET 的自由摆放更漂亮，但网格制更容易连对、也更接近课本电路图的整洁感，对孩子友好）。
- 数据模型：`CircuitDoc = { components: [{id, type, position, rotation, params}], wires: [{id, from: 端点ref, to: 端点ref, waypoints}] }` → 编译为解算器的抽象网表（节点合并在编译期完成）。文档可 JSON 序列化（沙盒存档 + 关卡定义共用同一格式）。

### D4. 可视化映射

- 电流小点：速度 ∝ I（上限封顶），密度固定；方向 = 实际电流方向（负值反向），I≈0 不动。
- 电势颜色：全电路电势归一化 → 暖色(高) → 冷色(低) 渐变着色导线与端点；提供开关可切换显示（避免初期信息过载）。
- 灯泡亮度：辉光半径与不透明度 ∝ P/P额定（10 级渐变 + SVG feGaussianBlur 辉光）；过功率 → 烧毁动画 + 灯丝断裂状态。
- 数值读数：电压表/电流表指针旋转 + 数字显示；悬停任意元件浮出 U/I/P 卡片（高中章节用）。

### D5. 关卡系统：关卡 = JSON 声明式定义

```
Level = {
  id, chapter, title, 引导文案,
  palette: 可用元件及数量上限,
  preset: 预置电路(可选, CircuitDoc 片段, 可标记锁定不可动),
  goals: [判定条件],   // 声明式，如 {type:'lit', target:'lamp1', min:0.8}
  stars: 三档星级条件,  // 如 用件数、是否烧毁过元件
  hints: [提示序列]
}
```

- 判定条件为一组声明式谓词（元件点亮/亮度区间/电表读数区间/开关状态因果/未烧毁任何元件…），解算结果每次更新后评估。MVP 实现 6–8 种谓词类型即可覆盖 8 章关卡。
- 8 章 ≈ 24–32 关（每章 3–4 关），内容顺序：① 点亮它（回路/断路/短路）② 串联 ③ 并联 ④ 欧姆定律与电表 ⑤ 滑动变阻器 ⑥ 电功率 ⑦ 电源内阻 ⑧ 动态分析。
- 进度（通关/星级）存 localStorage，版本化 key 防止格式升级冲突。

### D6. 美术：手写 SVG 组件，风格对齐定调图

- 每个元件 = 一个 React SVG 组件，含状态 props（如灯泡 brightness、开关 closed、保险丝 blown）。
- 风格语言：黏土玩具质感（圆角、柔和渐变、高光）、电池/灯泡/电表带表情，配色取自定调图（橙/黄/藏蓝/奶白/红）。
- 不用 AI 位图当运行素材（端点不精确、缩放糊、状态难做）；定调图仅作美术参考。

### D7. 部署：Cloudflare Pages，直连 git 或 wrangler 手动上传

- 纯静态产物（`dist/`），无环境变量、无 Functions。
- 仓库后续 `git init` + 推 GitHub 后可开自动部署；MVP 期 `npx wrangler pages deploy dist` 手动发布即可。

## Risks / Trade-offs

- [解算器边界情形出错（孩子必然乱接）] → 单测覆盖：短路、断路、孤岛、桥式、多电源、电表串反/并反、保险丝连锁熔断；与 Falstad 对拍关键用例。
- [网格连线体验做不顺手（拖拽是游戏手感的生命线）] → 最早做画布可玩原型（任务排序上画布+解算先行），真机（iPad 触摸）尽早试。
- [信息过载：三种可视化同时开孩子看花眼] → 分层默认：初中章节默认只开电流小点，电势颜色作为"透视镜"按钮引入；沙盒全开关自由切换。
- [电势颜色梯度在多电源/孤岛电路中无统一参考点] → 每个连通子图独立归一化；孤岛灰色显示"未接通"。
- [8 章关卡内容量大，拖累 MVP] → 关卡引擎（谓词判定）先做完整，首发填前 5 章内容（第 5 章滑动变阻器为初中必学，且第 4 章欧姆定律实验依赖它），第 6–8 章增量补充（JSON 数据活，不动代码）。
- [触摸与鼠标双输入的拖拽兼容] → 统一用 Pointer Events，开发期两种设备各过一遍冒烟清单。

## Open Questions

- 游戏正式名称（暂名"电路乐园"）与域名。
- 电流方向教学口径：初中课本用"正电荷定向移动方向"（正极→负极），小点按此方向流动；是否在高中章节提"电子实际反向"？（倾向：加一个彩蛋开关）
- 灯泡额定值参数怎么标（课本口径"2.5V 0.3A"式标注）——待关卡内容编写时定。
